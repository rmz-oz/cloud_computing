Topic 3: Docker Basic Commands

1 · Container Registries & Images

Docker images are stored in container registries. The default public registry is Docker Hub (hub.docker.com). Companies use private registries (AWS ECR, Azure Container Registry) for proprietary images.

docker pull nginx           # pull from Docker Hub (default registry)
docker pull nginx:1.23      # pull a specific version (tag)
docker images               # list all locally stored images
Image tags: nginx = nginx:latest. Always specify a version tag in production to avoid unexpected updates when rebuilding.

2 · Running Containers

docker run nginx                        # run container (foreground)
docker run -d nginx                     # run in detached (background) mode
docker run --name web nginx             # run with custom name
docker run -d --name web nginx          # detached + named
docker run -p 8080:80 nginx             # port binding: host:container
docker run -d -p 8080:80 --name web nginx  # all combined

Key docker run flags

FlagPurposeExample
-dDetached mode — runs container in backgrounddocker run -d nginx
--nameAssign a custom name to the containerdocker run --name web nginx
-pPort binding: hostPort:containerPortdocker run -p 8080:80 nginx
-eSet environment variablesdocker run -e MY_VAR=value nginx
-itInteractive terminal (for shells)docker run -it ubuntu bash

3 · Port Binding

By default, a container is isolated — its ports are NOT accessible from the host. Use -p to bind a host port to a container port:

docker run -p 8080:80 nginx
# Host port 8080 → Container port 80
# Access via: http://localhost:8080

docker run -p 3000:3000 node-app
# Same port for host and container (common for dev)
Format: -p HOST_PORT:CONTAINER_PORT. The container's internal port must match what the application listens on. Multiple ports: -p 80:80 -p 443:443

4 · Container Lifecycle Commands

docker ps               # list running containers
docker ps -a            # list all containers (including stopped)
docker stop web         # gracefully stop container named 'web'
docker start web        # start a stopped container
docker restart web      # stop + start

docker rm web           # remove a stopped container
docker rm -f web        # force remove a running container

docker rmi nginx        # remove an image
docker rmi nginx:1.23   # remove a specific image tag
docker run vs docker start: docker run creates a new container from an image. docker start restarts an already-created (but stopped) container without creating a new one.

5 · Inspecting Containers

docker logs web             # view logs of container 'web'
docker logs -f web          # follow (tail) logs in real time
docker logs --tail 100 web  # last 100 lines of logs

docker exec -it web bash    # open a bash shell inside running container
docker exec web ls /app     # run a single command inside container

docker inspect web          # detailed JSON info about container
docker stats                # live resource usage (CPU, RAM) for all containers
docker exec -it container bash — the most useful debugging command. Opens an interactive shell inside a running container so you can inspect files, check processes, and run commands.

6 · Image Management

docker images               # list images with REPOSITORY, TAG, IMAGE ID, SIZE
docker image ls             # same as docker images
docker rmi image_id         # remove image by ID
docker image prune          # remove all dangling (unused) images
docker system prune         # remove all stopped containers, unused images, volumes

docker tag myapp:latest myapp:v1.0    # tag an image
docker push myrepo/myapp:v1.0         # push to registry

7 · Pushing to a Private Registry (AWS ECR)

To push an image to AWS ECR (Elastic Container Registry):

# Step 1: Authenticate Docker to ECR
aws ecr get-login-password --region us-east-1 | \
  docker login --username AWS --password-stdin \
  123456789.dkr.ecr.us-east-1.amazonaws.com

# Step 2: Tag image for ECR
docker tag myapp:latest \
  123456789.dkr.ecr.us-east-1.amazonaws.com/myapp:latest

# Step 3: Push to ECR
docker push 123456789.dkr.ecr.us-east-1.amazonaws.com/myapp:latest
